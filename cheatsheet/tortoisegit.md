# TortoiseGit チートシート

## キーボード化テク
- `Shift+F10`：右クリックメニューをキーボードで出す
- メニュー項目の下線文字で選択（例：`C` = Commit）
- Commit ダイアログ：`Ctrl+Enter` で実行

## 推奨設定
- **"Push after commit" を常時ON** にする（commit→push が1ダイアログで完結）
- これだけで最大の時短

## Commit前チェックリスト（毎回）
1. 差分ファイル数が想定通りか
2. 各ファイルをダブルクリックして diff 確認
3. 改行コード/文字コードに変化がないか（VSCode右下ステータスバー確認）

## 全ファイル「削除/変更」が出た時
1. **commitしない**。ダイアログを閉じる
2. ファイル実在チェック（Explorerで目視）
3. パターンA（実在するが変更扱い）：改行/文字コード変換の可能性 → 先輩相談
4. パターンB（消えてる）：右クリック → "Revert..." で working tree を HEAD に戻す
5. 戻せない時：`git reflog`（"Show Reflog" メニュー）で過去のHEAD確認

## 緊急時：本番反映済みの修正を戻す
- 該当commit右クリック → **"Revert this revision"** → push
- これで該当commit前の状態に戻る

## コミットメッセージ規約
- `案件番号_更新内容`
- 案件番号は `Win+V`（クリップボード履歴）から貼り付け
- 「Recent messages」で過去パターン再利用
